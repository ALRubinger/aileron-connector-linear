# aileron-connector-linear

[Aileron](https://github.com/ALRubinger/aileron) connector for [Linear](https://linear.app). API-key auth, sandboxed to `api.linear.app:443`, governance metadata (idempotency + per-call approval gates) defined per operation.

This is the first greenfield connector built on top of [`aileron-codegen`](https://github.com/ALRubinger/aileron-codegen): the action manifests in `actions/` are emitted from `codegen/schema.graphql` + `codegen/gen.yaml`. Tracking issues: [aileron#915](https://github.com/ALRubinger/aileron/issues/915) (this connector) and [aileron#893](https://github.com/ALRubinger/aileron/issues/893) (the codegen umbrella).

## Status

`v0.0.x` — scaffolding. The connector WASM compiles, the action manifests are generated and committed, but the dispatch handlers in `connector/main.go` return `not_implemented` until the GraphQL-call layer lands. Tracking: [#915](https://github.com/ALRubinger/aileron/issues/915).

## Action surface

| Action | Op | Idempotent | Approval | Capability |
|---|---|---|---|---|
| [`actions/get-issue`](actions/get-issue/action.md) | `issue` | yes | none | `get_issue` |
| [`actions/viewer`](actions/viewer/action.md) | `viewer` | yes | none | `get_viewer` |
| [`actions/issue-create`](actions/issue-create/action.md) | `issue_create` | no | required | `create_issue` |
| [`actions/comment-create`](actions/comment-create/action.md) | `comment_create` | no | required | `create_comment` |

Writes (`issue-create`, `comment-create`) are gated on per-call user approval per ADR-0009; the runtime asks the user before dispatching, on denial nothing reaches Linear.

## Development

```sh
task build           # WASM build → connector.wasm
task generate        # Re-run aileron-codegen → actions/
task generate:check  # Verify committed actions/ matches codegen output (CI guard)
task test            # Unit tests + wasip1 build smoke
task vet             # go vet
task ci              # Full local gate (generate:check + test + vet)
```

`task generate` invokes `aileron-codegen` via `go run github.com/ALRubinger/aileron-codegen/cmd/aileron-codegen@latest`. The pin will switch to a tagged version once the overlay schema stabilizes (gate: two greenfield connectors shipped without overlay breaking changes — this connector is the first; see [aileron#893](https://github.com/ALRubinger/aileron/issues/893)).

## Auth + sandbox

- **Credential:** API key (`api_key` kind). User issues a key at [linear.app/settings/api](https://linear.app/settings/api) and binds it once via `aileron binding setup`. The runtime injects `Authorization: <api_key>` host-side on every outbound request marked `credential: "api_key"`; the connector binary never sees the value.
- **Network:** Sandboxed to `api.linear.app:443` only. Anything else is denied at the WASM boundary (ADR-0005).
