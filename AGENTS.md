# AGENTS.md

Operator notes for AI agents (and humans) working in this repo. Read this first.

## What this repo is

`aileron-connector-linear` is an [Aileron](https://github.com/ALRubinger/aileron) connector for [Linear](https://linear.app). It is the first greenfield connector built on top of [`aileron-codegen`](https://github.com/ALRubinger/aileron-codegen); its presence (and overlay stability) is part of the codegen umbrella's acceptance criterion.

**Tracking issues:**

- [ALRubinger/aileron#915](https://github.com/ALRubinger/aileron/issues/915) — this connector. Open questions, proposed action set, follow-up work.
- [ALRubinger/aileron#893](https://github.com/ALRubinger/aileron/issues/893) — the codegen umbrella that this connector validates.

Read both before starting work.

## Local development

```sh
task build           # WASM build → connector.wasm
task generate        # Re-run aileron-codegen → actions/
task generate:check  # Verify committed actions/ matches codegen
task test            # Unit tests + wasip1 build smoke
task vet             # go vet
task ci              # Full local gate
```

`task ci` mirrors the GitHub Actions workflow. Run it locally before pushing.

## Generated files are committed

`actions/<name>/action.md` is **generated by `aileron-codegen`** from `codegen/schema.graphql` + `codegen/gen.yaml`. The generated tree is **committed** so PRs are reviewable end-to-end.

Workflow:

1. Edit `codegen/schema.graphql` or `codegen/gen.yaml`.
2. Run `task generate`.
3. Review the resulting `actions/` diff like any other code change.
4. Commit both the overlay edit and the regenerated `actions/`.

The CI `generate:check` job runs `task generate` and `git diff --exit-code -- actions`. If the committed `actions/` drifts from what codegen would emit, CI fails. **Do not hand-edit `actions/<name>/action.md`** — your edit will not survive the next `task generate`. Put per-action prose somewhere the codegen prose emitter can consume (TBD) or override via `intent` / `action_name` / `execute_id` in `gen.yaml`.

## Codegen version pin

`Taskfile.yml`'s `generate` target invokes:

```
go run github.com/ALRubinger/aileron-codegen/cmd/aileron-codegen@latest ...
```

This is `@latest` while `aileron-codegen` is pre-release. The pin moves to a tagged semver once the overlay schema stabilizes (per [aileron#893](https://github.com/ALRubinger/aileron/issues/893) acceptance: two greenfield connectors shipped without overlay breaking changes — this connector is the first).

If you re-run `task generate` and the output drifts unexpectedly, check whether `aileron-codegen`'s `main` branch shifted behavior — `git log` on that repo will tell you.

## WASM target

The connector lives under `connector/` as its own Go module (`github.com/ALRubinger/aileron-connector-linear/connector`). `connector/main.go` builds for `GOOS=wasip1 GOARCH=wasm` and reads `{"op": "...", "args": {...}}` from stdin, writing `{"output": {...}}` or `{"error": {"class": "...", "message": "..."}}` to stdout. The runtime exec's the WASM, pipes stdin in, captures stdout.

The shipped dispatch is a **stub** — every known op returns `not_implemented`. The Linear GraphQL POST layer + per-op response shaping ships separately ([#915](https://github.com/ALRubinger/aileron/issues/915) follow-up PR).

When the dispatch handlers land, they will use Aileron's host imports for HTTP (the runtime injects the API key host-side at the network boundary — the connector binary never holds it; see ADR-0005). The Google connector ([`aileron-connector-google/connector/main.go`](https://github.com/ALRubinger/aileron-connector-google/blob/main/connector/main.go)) is the reference implementation.

## Op naming

The dispatch op names in `connector/main.go` must match the `op` field emitted into each `action.md`'s `[[execute]]` block. Codegen derives `op = snake_case(operationId)`, so:

| GraphQL field | `op` |
|---|---|
| `issue` | `issue` |
| `viewer` | `viewer` |
| `issueCreate` | `issue_create` |
| `commentCreate` | `comment_create` |

If you add an op, regenerate `actions/` first (`task generate`), check the emitted `op` value, then add the dispatch case.

## Workflow conventions

- **Branch + PR for everything except README.** Conventional commit titles (`feat:`, `fix:`, `docs:`, `refactor:`, `test:`).
- **Squash-merge with `--admin --delete-branch`.** The repo is admin-merge-authorized.
- **Wait for CI before merging** — even with `--admin`, never merge while a check is pending.
- **No backwards-compat shims.** Pre-release.
- **No emojis in code or docs unless explicitly requested.**

## ADRs

All ADRs live in [`ALRubinger/aileron/docs/adr/`](https://github.com/ALRubinger/aileron/tree/main/docs/src/content/docs/adr) and are referenced by number. The ones this connector touches most:

- **ADR-0002** — connector identity + content-addressed hash + `0.0.0-dev` placeholder substitution.
- **ADR-0005** — sandbox + credential mediation (the `Authorization` header is injected host-side; connector never sees the key).
- **ADR-0009** — per-call user approval gating (`[approval]` block in action.md, runtime asks user before dispatching).
- **ADR-0010** — idempotency declaration + runtime retry semantics.
